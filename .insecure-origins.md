# HTTP ile Mikrofon ve Bildirim İzinleri

Tarayıcılar güvenlik nedeniyle mikrofon (`getUserMedia`) ve bildirim (`Notification`) API'lerini yalnızca **güvenli bağlamlarda** (HTTPS veya localhost) etkinleştirir.

Docker'da HTTP ile çalıştırırken bu özellikleri kullanabilmek için tarayıcı ayarlarını değiştirmeniz gerekiyor.

## Chrome İçin Ayar

### Geçici Çözüm (Flag ile)

1. Chrome'u kapatın
2. Aşağıdaki komutu çalıştırın:

**Windows:**
```bash
"C:\Program Files\Google\Chrome\Application\chrome.exe" --unsafely-treat-insecure-origin-as-secure="http://YOUR_SERVER_IP:8890"
```

**Mac:**
```bash
open -na "Google Chrome" --args --unsafely-treat-insecure-origin-as-secure="http://YOUR_SERVER_IP:8890"
```

### Kalıcı Çözüm (chrome://flags)

1. Chrome'da `chrome://flags` adresine gidin
2. `Insecure origins treated as secure` araması yapın
3. Kutucuğa sunucu adresinizi ekleyin: `http://YOUR_SERVER_IP:8890`
4. Chrome'u yeniden başlatın

## Firefox İçin Ayar

1. Adres çubuğuna `about:config` yazın
2. "I accept the risk" uyarısını kabul edin
3. Aşağıdaki ayarları arayın ve değiştirin:

```
media.devices.insecure.enabled = true
media.getusermedia.insecure.enabled = true
```

## Edge İçin Ayar

Chrome ile aynı yöntemi kullanın (`edge://flags`).

## Kalıcı Çözüm: HTTPS Kurulumu

Üretim ortamında kesinlikle HTTPS kullanmalısınız. Bu için:

1. **Reverse Proxy (Nginx/Traefik)**: Önünde SSL sertifikalı bir reverse proxy kullanın
2. **Let's Encrypt**: Ücretsiz SSL sertifikası alın
3. **Self-Signed Certificate**: Test için kendi sertifikanızı oluşturun

### Örnek Nginx Konfigürasyonu

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    location / {
        proxy_pass http://localhost:8890;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```
